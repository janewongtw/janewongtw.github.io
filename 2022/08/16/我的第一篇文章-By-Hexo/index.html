變數及常數

為什麼我們需要變數及常數？

試著想像一下，如果每個人都沒有名字，該如何分辨誰是誰呢？

總不能每個人都叫欸吧😂（欸欸你過來一下）

變數Variable

變數就像標籤一樣，貼在某個物件身上，本身是沒有型態，也無法單獨存在

cat是一個變數，貼在”kitty”這個字串身上，要使用變數的話，要讓它指向某個東西（字串或是數字之類的）

根據變數的作用範圍分為四種類型

區域變數 local Variable

定義在method裡面，無法在method以外的地方使用

以小寫字母為開頭（name, title）

沒有預設值

全域變數 instance Variable

能編寫於任何地方，全域能使用導致除錯難度增高

開頭為符號$（$name, $title）

預設值為nil

實體變數 class Variable

在同一個object中，所有methods都能使用

開頭為符號@（@name, @title）

預設值為nil

類別變數 global Variable

整個class所創造出的實體都可以共用

開頭為符號@@（@@name, @@title）

沒有預設值

變數的命名必須清楚有意義，好好取名字非常重要

為了讓程式能夠指向正確的對象，給予一個字串或數字有意義的名字，自己使用或和別人協作會更容易知道到底在做什麼

分享一下寫code初期我常犯的錯誤😥

取名字是個學問，初期不知道該如何命名的時候，都會把a、b或是x、y之類的字母拿來使用，如果程式碼很少的話，稍微回想一下，還能知道當初寫這一行到底在做什麼，隨著學習到的功能和程式碼日漸徒增，那些無意義的單字根本不可能讓你想起為什麼會這樣寫啊~~~~~(除非你有寫註解)

雖然這個method命名簡單又暴力，但約略知道這個方法在做什麼事

ruby一次可以指定多個變數

ruby裡面的 | | 有點像是預設值的概念，如果裡面的值判斷true且成立的話，就會給你這個結果

常數Constant

開頭一定要是大寫字母，在ruby的世界裡，常數是可以被修改的

所有的class、module的命名方式都必須是常數

定義方法 def

定義類別 class
